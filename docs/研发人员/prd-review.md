# 移动端图像直方图分析系统 PRD 评审记录

## 1. 文档说明

本文档记录研发人员对需求规格、设计文档、测试计划和课程验收要求的评审结论。评审目标不是重新定义需求，而是确认需求是否能落到当前 Android WebView + H5 Canvas 技术方案中，并明确研发实现边界。

## 2. 评审信息

| 项目 | 内容 |
| --- | --- |
| 项目名称 | 移动端图像直方图分析系统 |
| 评审角色 | 需求分析、研发、测试、项目管理 |
| 评审对象 | 需求规格说明书、设计文档、使用说明、测试计划、项目计划 |
| 当前阶段 | 主功能完成后的文档反补与交付收口 |
| 评审结论 | 核心需求可实现且已基本实现，后续重点是证据回填和答辩材料整理 |

## 4. 范围评审

### 4.1 保留范围

研发侧确认保留以下核心范围：

- Android APK；
- WebView 加载本地 H5；
- 本地图片选择；
- Canvas 像素读取；
- 灰度公式计算；
- 256 bin 统计；
- `0-100` 归一化；
- `256x100` 黑白直方图；
- 生成耗时展示；
- baseline/optimized 性能对照；
- 演示辅助指标和日志。

### 4.2 不纳入范围

以下内容不进入当前版本：

- 用户登录；
- 后端接口；
- 数据库；
- 云端图片上传；
- 远程部署；
- 多用户协作；
- 网络图片处理；
- 为了好看替换课程指定公式或输出规则。

## 5. 技术方案评审

| 方案点 | 评审结论 |
| --- | --- |
| Android WebView 壳 | 合适。能以 APK 形态交付，同时降低原生图像处理开发成本 |
| H5 Canvas | 合适。能读取 RGBA 像素并绘制固定尺寸直方图 |
| 原生 Java 保存桥 | 可接受。只用于用户确认后的拼接图保存，不影响核心算法 |
| Product Flavors | 合适。baseline 和 optimized 可并装，便于性能对照 |
| 不引入前端框架 | 合适。项目规模小，原生 H5 更直接 |
| 不引入后端 | 必须坚持。课程验收不需要后端，反而增加风险 |

## 6. 风险评审

| 风险 | 影响 | 处理结论 |
| --- | --- | --- |
| 真机数据未回填 | 测试报告证据不完整 | 由测试人员补充机型、Android 版本、截图和耗时 |
| README 局部旧路径 | 文档索引可能误导 | 后续可统一修正文档索引，不影响源码实现 |
| 超大图片性能波动 | 现场演示不稳定 | 答辩优先使用准备好的中等尺寸图片 |
| 保存图片权限差异 | 增强功能可能失败 | 不作为核心验收路径 |
| 功能继续扩张 | 影响收尾稳定性 | 当前阶段冻结功能，只修阻塞问题 |

## 7. 验收口径评审

研发侧建议最终验收按以下顺序展示：

```text
安装或打开 optimized APK
-> 断网说明
-> 选择本地图片
-> 展示原图预览
-> 查看 256x100 直方图
-> 查看耗时和指标
-> 解释灰度公式、归一化和性能对照
```

baseline APK 只在需要说明性能优化时打开，不作为主演示版本。

## 8. 评审结论

评审认为当前需求范围清晰，技术方案可控，核心功能已经实现并有研发自测证据支撑。研发侧后续不建议继续扩大功能范围，应重点配合测试人员补齐真机验证信息，并配合答辩材料整理源码、算法和性能对照说明。
