# 研发任务清单

## 1. 文档说明

本文档用于把“移动端图像直方图分析系统”的需求拆分为可执行任务，并记录每个任务的推进状态。后续每次完成任务、任务范围发生变化、或遇到阻塞时，都必须同步更新本文档。

最后更新日期：2026-07-08

## 2. 状态规则

| 状态 | 含义 | 更新要求 |
| --- | --- | --- |
| `todo` | 尚未开始 | 保留任务目标、依赖和验收标准 |
| `doing` | 正在推进 | 写明当前处理内容或关联提交 |
| `done` | 已完成 | 写明完成结果、关键文件或验证方式 |
| `blocked` | 暂时阻塞 | 写明阻塞原因、等待事项和下一步处理方式 |
| `deferred` | 后续单独推进 | 写明为什么不进入当前主流程 |

更新原则：

- 完成任一任务后，将状态更新为 `done`，并补充完成说明。
- 任务被阻塞时，将状态更新为 `blocked`，并写清阻塞原因。
- 若新增、拆分或合并任务，需要同步维护任务编号、依赖关系和验收标准。
- 性能优化任务保持独立，不作为第一阶段主流程实现的阻塞条件。
- 不得把尚未实现、尚未测试或尚未打包的内容标记为 `done`。
