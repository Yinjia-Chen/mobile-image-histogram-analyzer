# 构建运行与发布说明

## 1. 文档说明

本文档面向研发人员，说明项目本地预览、自动化验证、Android 构建、APK 产物和发布前检查流程。

## 2. 环境要求

| 项目 | 要求 |
| --- | --- |
| Node.js | 当前使用 v24.16.0 |
| npm | 随 Node.js 使用 |
| Android Studio | 提供 JBR 和 Android SDK |
| Java | 使用 Android Studio 自带 JBR |
| Android SDK | 已安装对应 compileSdk / targetSdk |

Android 构建示例环境变量：

```bash
JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
ANDROID_HOME="/Users/framia/Library/Android/sdk"
```

## 3. 本地预览

本地浏览器预览用于调试 H5 页面，不作为最终交付方式。

```bash
npm run preview
```

打开：

| 页面 | 地址 |
| --- | --- |
| 正式演示版本 | `http://127.0.0.1:4173/index.html` |
| 低效压力对照版本 | `http://127.0.0.1:4173/index-baseline.html` |

说明：

- 浏览器预览方便 UI 和截图；
- 正式验收仍以 Android APK 离线运行为准；
- 浏览器环境不能完全等价于 Android WebView。

## 4. 自动化验证

常用命令：

```bash
npm run test:histogram
npm run test:offline
npm run test:baseline
npm run benchmark:histogram
npm run check:source-comments
npm run harness:verify
```

| 命令 | 作用 |
| --- | --- |
| `test:histogram` | 算法和主流程 fixture |
| `test:offline` | 离线 assets 和 Android 入口检查 |
| `test:baseline` | 低效对照版本与当前优化版本一致性 |
| `benchmark:histogram` | 性能对照数据 |
| `check:source-comments` | 源码注释覆盖检查 |
| `harness:verify` | 项目综合验证 |

## 5. Android 构建

### 5.1 构建低效对照版本和正式演示版本 APK

```bash
JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home" \
ANDROID_HOME="/Users/framia/Library/Android/sdk" \
./gradlew --offline assembleBaselineDebug assembleOptimizedDebug
```

### 5.2 Flavor 说明

| Flavor | 输出 | 用途 |
| --- | --- | --- |
| `baselineDebug` | 低效压力对照版本 debug APK | 性能对照 |
| `optimizedDebug` | 正式演示版本 debug APK | 当前优化版本主演示 |

`app/build.gradle` 使用 `BuildConfig.LOCAL_ENTRY_URL` 区分两种入口：

| Flavor | 入口 |
| --- | --- |
| baseline | `file:///android_asset/index-baseline.html` |
| optimized | `file:///android_asset/index.html` |

## 6. 产物整理

最终演示常用产物：

| 产物 | 路径 |
| --- | --- |
| 正式演示版本 APK | `dist/mobile-histogram-optimized-coinstall-debug.apk` |
| 低效压力对照版本 APK | `dist/mobile-histogram-baseline-coinstall-debug.apk` |
| 基础阶段 APK | `dist/mobile-histogram-stage4-debug.apk` |

重新打包后需要更新：

- APK 大小；
- SHA256；
- 包名；
- versionName；
- H5 入口；
- 构建命令和执行结果；
- 对应测试记录。

## 7. APK 检查建议

### 7.1 内容检查

检查 APK 内是否包含：

```text
assets/index.html
assets/app.js
assets/index-baseline.html
assets/baseline-app.js
assets/style.css
AndroidManifest.xml
classes.dex
resources.arsc
```

### 7.2 入口检查

baseline APK 应加载：

```text
file:///android_asset/index-baseline.html
```

optimized APK 应加载：

```text
file:///android_asset/index.html
```

### 7.3 包名检查

| APK | 包名 |
| --- | --- |
| baseline | `com.framia.mobilehistogram.baseline` |
| optimized | `com.framia.mobilehistogram.optimized` |

## 8. 发布前检查清单

| 检查项 | 要求 |
| --- | --- |
| 算法测试 | `npm run test:histogram` 通过 |
| 离线检查 | `npm run test:offline` 通过 |
| 对照一致性 | `npm run test:baseline` 通过 |
| 综合验证 | `npm run harness:verify` 通过 |
| APK 构建 | baseline 和 optimized 均构建成功 |
| APK 哈希 | 更新到测试记录 |
| README 口径 | 与最新 APK 和文档一致 |
| 任务台账 | 完成或阻塞状态已同步 |
| 真机验证 | 手机型号、Android 版本、截图和耗时待测试人员回填 |

## 9. 常见问题

| 问题 | 处理方式 |
| --- | --- |
| Gradle 找不到 Java | 设置 `JAVA_HOME` 为 Android Studio JBR |
| 构建依赖无法下载 | 使用已有离线 Gradle 缓存；如必须联网需单独确认 |
| APK 无法并装 | 检查 flavor 是否带不同 `applicationIdSuffix` |
| 页面打开白屏 | 检查入口 URL、assets 是否打包、WebView 背景和启动页配置 |
| 文件选择无响应 | 检查 `onShowFileChooser` callback 是否被正确结束 |
| 保存图片失败 | 检查 MediaStore 写入逻辑和系统权限策略 |

## 10. 结论

研发构建流程已经形成闭环：H5 可本地预览，算法可脚本验证，APK 可通过 flavor 输出低效压力对照版本和正式演示版本两份产物。后续每次涉及源码或 APK 的修改，都应同步更新测试证据和任务台账。
