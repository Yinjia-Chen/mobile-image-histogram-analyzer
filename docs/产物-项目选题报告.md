# 项目背景以及功能说明

## 移动端图像直方图分析系统

| 项目项 | 说明 |
| --- | --- |
| 选题来源 | 软件开发实践课程任务书：课题 1“图像直方图计算及性能优化”。 |
| 项目形态 | Android WebView 壳 App，最终交付 Android APK 安装包。 |
| 核心技术 | Android WebView、HTML5、CSS、JavaScript、Canvas。 |
| 运行方式 | 安装到 Android 手机后离线运行，无需后端服务和网络连接。 |

## 一、项目背景

数字图像在移动设备、智能终端和软件测试场景中被广泛使用。图像直方图是描述图像亮度分布的重要工具，可以反映一张图片中不同灰度值的像素数量分布情况。通过观察直方图，用户能够快速判断图像整体偏亮、偏暗、对比度是否集中等基本特征。

课程任务书要求开发一个移动端程序，上传或选择图像后统计彩色图像中的像素，并生成 `256x100` 的黑白直方图，同时显示直方图生成速度。该任务既要求结果正确，也关注移动端环境下的计算性能，因此适合作为图像处理、移动端开发和性能优化综合实践项目。

结合团队对开发难度、验收稳定性和答辩展示效果的综合判断，本项目选择“图像直方图计算及性能优化”作为实践选题，并采用 Android WebView + H5 Canvas 的混合开发路线。该路线能够以 APK 形式交付，具备真实移动端应用形态，同时将核心算法集中在前端 Canvas 和 JavaScript 中，降低开发复杂度，便于快速实现和测试。

## 二、项目目标

本项目拟开发一款可离线运行的移动端图像直方图分析系统。用户安装 APK 后，可以从手机本地选择图片，系统在本地完成图像像素读取、灰度化、直方图统计、归一化处理和直方图绘制，并在界面中展示计算耗时。

- 完成任务书要求的 `256x100` 黑白直方图生成。
- 使用指定灰度化公式保证统计结果具有一致性。
- 在手机本地完成图像处理，减少网络依赖并保护用户图片隐私。
- 先保证主流程功能稳定可用，再根据实测结果进行启发式、渐进式性能优化。
- 形成可安装、可演示、可测试、可答辩的课程实践作品。

## 三、功能说明

系统主要面向课程验收和移动端图像处理实践场景，围绕“选择图片、生成直方图、显示耗时、展示结果”形成完整功能闭环。核心功能如下表所示。

| 序号 | 功能模块 | 功能说明 |
| --- | --- | --- |
| 1 | 图片选择 | 支持从手机本地相册选择待分析图片，作为直方图计算的输入数据。 |
| 2 | 原图预览 | 在应用界面中展示用户选择的图片，便于确认当前处理对象。 |
| 3 | 灰度化处理 | 按照题目指定公式 `gray = red * 0.299 + green * 0.587 + blue * 0.114` 将彩色像素转换为灰度值。 |
| 4 | 直方图统计 | 统计 `0-255` 共 256 个灰度值的像素出现次数，形成原始灰度分布数据。 |
| 5 | 结果归一化 | 将统计结果按最大像素数量等比例归一化到 `0-100`，用于生成 `256x100` 的直方图图像。 |
| 6 | 直方图绘制 | 使用 H5 Canvas 绘制 `256x100` 黑白直方图，横轴对应灰度值，纵向高度对应归一化后的像素数量。 |
| 7 | 耗时显示 | 记录并展示直方图生成耗时，用于验证移动端本地计算性能。 |
| 8 | 渐进式性能优化 | 在主流程稳定后，保留优化前后耗时记录，便于在测试报告和答辩中说明优化过程。 |

## 四、系统运行方式

系统采用 Android APK 形式交付。Android 原生层主要负责应用封装、WebView 初始化、本地页面加载和图片选择能力；H5 层负责页面展示、Canvas 图像处理和直方图算法实现。安装完成后，用户点击手机桌面图标即可进入应用。

由于所有图像处理逻辑均在手机本地执行，系统不依赖服务器、数据库或远程接口。验收时只需要提前安装 APK，并在手机中准备测试图片，即使手机处于无网络状态，也可以完成核心功能演示。

## 五、图像处理流程

本系统的核心处理流程从用户选择图片开始，到直方图绘制和耗时展示结束。整体流程如下表所示。

| 步骤 | 处理环节 | 说明 |
| --- | --- | --- |
| 1 | 用户选择本地图片 | 从手机相册或文件选择图片，进入预览和处理流程。 |
| 2 | Canvas 读取像素 | WebView 中的 H5 页面将图片绘制到 Canvas，并读取 RGBA 像素数据。 |
| 3 | 灰度值计算 | 遍历像素数组，根据指定公式计算每个像素的灰度值。 |
| 4 | 灰度直方图统计 | 使用长度为 256 的数组记录每个灰度值出现次数。 |
| 5 | 归一化与绘制 | 按最大计数归一化到 `0-100`，并绘制为 `256x100` 黑白直方图。 |
| 6 | 展示耗时与结果 | 在界面中显示生成耗时、原图预览和直方图结果。 |

## 六、项目范围说明

本阶段项目范围聚焦于课程任务书中的核心验收目标，优先保证功能完整、运行稳定、结果准确和文档规范。

- 范围内：本地图片选择、灰度化、直方图统计、归一化、Canvas 绘制、耗时展示、基础测试。
- 范围外：用户登录、云端图片存储、数据库管理、后端接口服务、微信小程序发布审核。
- 可扩展方向：渐进式性能优化、测试结果导出、历史记录、更多图像分析指标。

## 七、预期成果

项目完成后，团队将提交一个能够在 Android 手机上安装运行的 APK 应用，以及对应的源码、需求分析、概要设计、测试报告和使用说明。答辩展示时，团队可以通过本地图片演示直方图生成过程，并结合不同图片尺寸的测试结果说明系统性能表现。

本项目的主要价值在于：以较低的系统复杂度完成清晰可验收的移动端图像处理功能，同时能够体现算法实现、移动端适配、渐进式性能优化和工程文档编写等课程实践要求。
