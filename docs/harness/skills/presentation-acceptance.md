# Presentation Acceptance Skill

Use this when preparing final demo, acceptance checklist, defense script, or APK
handoff.

## Demo Path

The final demonstration should be short and repeatable:

1. Install or open the Android APK.
2. Confirm the app loads offline.
3. Choose a prepared local image.
4. Show the image preview.
5. Generate the `256x100` histogram.
6. Point out the displayed generation time.
7. Explain the grayscale formula, normalization, and optimization evidence.

## Prepared Materials

Keep at least three test images ready:

- small or medium normal photo;
- bright or dark image;
- larger image for performance comparison.

## Risk Guardrails

- Do not rely on a network image during defense.
- Do not demo only the desktop browser if the claimed deliverable is APK.
- Do not explain performance without a recorded measurement table.
- Prepare a fallback image and a known-good APK before final acceptance.
