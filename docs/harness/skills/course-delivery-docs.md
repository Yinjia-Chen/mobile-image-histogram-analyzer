# Course Delivery Docs Skill

Use this when creating or updating course documents, reports, usage guides,
testing material, or PPT content.

## Required Document Families

- project background and feature description;
- requirement analysis;
- outline design;
- implementation/source description;
- test plan, test cases, execution record, and test report;
- usage guide;
- defense PPT;
- course ideology report, if the task is about that deliverable.

## Consistency Guardrails

- Do not claim an APK, optimization, benchmark, or feature exists before it is
  implemented or evidenced.
- Keep terminology consistent: Android WebView shell, H5 Canvas, local image,
  grayscale histogram, `256x100`, `300ms`.
- Separate core scope from optional extensions.
- Tie tests to the course scoring points: accuracy, timing, stability, and
  explainability.

## Evidence Checklist

When writing docs, look for:

- source files that implement the described behavior;
- screenshots or demo notes;
- benchmark table rows;
- known limitations;
- team roles and deliverable ownership.
