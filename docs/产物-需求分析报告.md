## 2. 项目背景

数字图像在移动设备、智能终端和软件测试场景中被广泛使用。图像直方图是描述图像亮度分布的重要工具，可以反映一张图片中不同灰度值的像素数量分布情况。通过图像直方图，用户可以直观看到图片整体偏亮、偏暗、亮度集中或对比度分布等基本特征。

课程任务书要求开发一个移动端程序，上传或选择图像后统计彩色图像中的像素，并生成 `256x100` 的黑白直方图，同时显示直方图生成速度。该任务同时关注结果准确性和移动端计算性能，适合作为图像处理、移动端开发和性能优化的综合实践项目。

结合项目团队对开发难度、验收稳定性和答辩展示效果的判断，本项目采用 Android WebView + H5 Canvas 的混合开发路线。Android 层负责 APK 封装、WebView 加载和本地图片选择能力；H5 层负责图片预览、像素读取、灰度计算、直方图统计、归一化、绘制和耗时展示。

## 3. 项目目标

### 3.1 总体目标

开发一款可安装在 Android 手机上的离线图像直方图分析应用。用户安装 APK 后，可以选择手机本地图片，系统在手机本地完成图像处理，并展示 `256x100` 黑白直方图和生成耗时。

### 3.2 具体目标

| 编号 | 目标说明 | 优先级 |
| --- | --- | --- |
| OBJ-01 | 支持用户从手机本地选择图片并进行预览 | 高 |
| OBJ-02 | 按任务书指定公式完成 RGB 到灰度值的转换 | 高 |
| OBJ-03 | 统计 `0-255` 共 256 个灰度值的像素数量 | 高 |
| OBJ-04 | 将统计结果归一化到 `0-100` 的纵向范围 | 高 |
| OBJ-05 | 绘制 `256x100` 黑白直方图 | 高 |
| OBJ-06 | 显示完整直方图生成耗时，并尽量控制在 `300ms` 以内 | 高 |
| OBJ-07 | 支持基础实现与优化实现的性能对比，为测试报告和答辩提供证据 | 中 |
| OBJ-08 | 保证核心功能离线可用，不依赖后端、数据库或网络 | 高 |

## 4. 项目范围

### 4.1 范围内需求

本项目范围聚焦课程任务书中的核心验收目标，主要包括：

- Android APK 安装包交付。
- Android WebView 本地页面加载。
- 手机本地图片选择。
- 图片预览。
- Canvas 读取图片像素。
- 指定公式灰度化。
- `0-255` 灰度值统计。
- `0-100` 归一化。
- `256x100` 黑白直方图绘制。
- 直方图生成耗时展示。
- 性能测试、准确性测试和稳定性测试。
- 项目需求、设计、测试、使用说明和答辩材料编写。

### 4.2 范围外需求

以下内容不属于本阶段核心范围，不应影响核心验收路径：

- 用户登录与账号体系。
- 云端图片上传与存储。
- 数据库管理。
- 后端接口服务。
- 远程部署平台。
- 微信小程序账号、审核与发布。
- 社交分享、历史云同步等非核心扩展。

### 4.3 可选扩展

在核心功能稳定后，可以考虑以下扩展，但不得替代或影响核心需求：

- 不同算法模式的耗时对比。
- 测试结果导出。
- 历史处理记录。
- 更多图像统计指标展示。
- 对不同尺寸图片的自动测试分组。

## 6. 总体业务流程

```text
启动 Android APK
    ↓
WebView 加载本地 H5 页面
    ↓
用户选择手机本地图片
    ↓
页面展示原图预览
    ↓
Canvas 绘制图片并读取 RGBA 像素数据
    ↓
按指定公式计算灰度值
    ↓
统计 0-255 灰度值出现次数
    ↓
按最大计数归一化到 0-100
    ↓
绘制 256x100 黑白直方图
    ↓
展示直方图和生成耗时
```

## 7. 功能需求

### 7.1 功能需求列表

| 编号 | 功能名称 | 需求描述 | 优先级 | 验收标准 |
| --- | --- | --- | --- | --- |
| FR-01 | 应用启动 | 系统应以 Android APK 形式安装并启动，进入图像直方图分析主界面 | 高 | 安装后可从桌面图标打开应用 |
| FR-02 | 本地页面加载 | Android WebView 应加载本地 H5 页面，不依赖远程页面资源 | 高 | 无网络状态下可进入主界面 |
| FR-03 | 图片选择 | 系统应支持用户从手机本地选择一张图片作为输入 | 高 | 用户可从相册或文件入口选中图片 |
| FR-04 | 图片预览 | 系统应在界面中展示用户选择的原始图片预览 | 中 | 选图后可看到当前处理对象 |
| FR-05 | 像素读取 | 系统应通过 Canvas 获取图片 RGBA 像素数据 | 高 | 后续灰度化可基于真实像素数据进行 |
| FR-06 | 灰度化计算 | 系统应使用 `gray = red * 0.299 + green * 0.587 + blue * 0.114` 计算灰度值 | 高 | 不得使用其他公式替代任务书公式 |
| FR-07 | 灰度值统计 | 系统应统计 `0-255` 共 256 个灰度值的像素数量 | 高 | 统计数组长度为 256，覆盖全部灰度值 |
| FR-08 | 结果归一化 | 系统应将灰度统计结果按最大计数等比例归一化到 `0-100` | 高 | 最大灰度计数对应高度 100，其余灰度按比例缩放 |
| FR-09 | 直方图绘制 | 系统应绘制 `256x100` 黑白直方图 | 高 | 横向 256 个灰度 bin，纵向最大高度 100 |
| FR-10 | 耗时显示 | 系统应显示直方图生成耗时 | 高 | 页面显示本次生成耗时，单位建议为 ms |
| FR-11 | 重新选择图片 | 系统应支持用户重新选择另一张图片并刷新结果 | 中 | 新图片处理后原结果被更新 |
| FR-12 | 异常提示 | 当用户未选择图片、选择失败或图片无法读取时，系统应给出明确提示 | 中 | 异常情况下页面不崩溃，并显示可理解提示 |
| FR-13 | 性能对比入口 | 系统可提供基础实现和优化实现的耗时对比模式 | 中 | 相同图片可记录不同实现模式下的耗时 |
| FR-14 | 离线演示 | 系统核心功能应在无网络环境下完成 | 高 | 断网状态下可选择本地图片并生成直方图 |

### 7.2 核心算法需求

| 编号 | 需求描述 | 验收重点 |
| --- | --- | --- |
| ALG-01 | 灰度化公式必须采用任务书指定公式 | 公式一致性 |
| ALG-02 | 灰度值应映射到 `0-255` 的整数区间 | 统计边界正确 |
| ALG-03 | 每个有效像素均应参与统计 | 不漏统计、不重复统计 |
| ALG-04 | 直方图数组必须包含 256 个 bin | bin 数量正确 |
| ALG-05 | 归一化结果应落在 `0-100` 范围内 | 输出高度合法 |
| ALG-06 | 绘制结果必须来源于真实统计数据，而非示意图 | 结果真实性 |

### 7.3 性能需求

| 编号 | 需求描述 | 验收重点 |
| --- | --- | --- |
| PERF-01 | 系统应记录完整直方图生成耗时 | 计时范围应覆盖像素读取、灰度计算、bin 统计、归一化和绘制 |
| PERF-02 | 直方图生成速度应尽量控制在 `300ms` 以内 | 使用指定测试设备和图片记录结果 |
| PERF-03 | 性能测试应保留图片尺寸、设备环境、实现模式和耗时数据 | 测试报告可复现 |
| PERF-04 | 基础实现与优化实现的耗时应在相同测试条件下比较 | 对比公平 |

## 9. 数据需求

### 9.1 输入数据

| 数据项 | 说明 | 要求 |
| --- | --- | --- |
| 本地图片 | 用户从手机本地选择的图片文件 | 应支持常见图片格式，如 JPEG、PNG |
| 图片像素数据 | Canvas 读取出的 RGBA 像素数组 | 应包含每个像素的 R、G、B、A 通道 |

### 9.2 中间数据

| 数据项 | 说明 | 要求 |
| --- | --- | --- |
| 灰度值 | 由 RGB 通道按指定公式计算得到 | 范围应为 `0-255` |
| 直方图计数数组 | 长度为 256 的数组，记录每个灰度值出现次数 | 每个索引对应一个灰度值 |
| 归一化数组 | 将计数数组按最大值缩放到 `0-100` 后得到 | 用于绘制直方图高度 |

### 9.3 输出数据

| 数据项 | 说明 | 要求 |
| --- | --- | --- |
| 黑白直方图 | `256x100` 的图像化统计结果 | 应与统计数据一致 |
| 生成耗时 | 本次直方图生成消耗时间 | 应显示在界面中，单位建议为 ms |
| 测试记录 | 测试图片、设备、模式、耗时等信息 | 用于测试报告与答辩说明 |

## 15. 约束与假设

### 15.1 约束

- 必须保留 Android APK 的最终交付形态。
- 必须保留 Android WebView + H5 Canvas 的核心技术路线。
- 必须使用任务书指定灰度化公式。
- 必须生成 `256x100` 黑白直方图。
- 必须显示直方图生成耗时。
- 不引入后端、数据库、登录、云存储等非核心范围。

### 15.2 假设

- 验收设备为常见 Android 手机或 Android 模拟器。
- 用户选择的图片为常见图片格式。
- 测试图片已提前保存在手机本地。
- 性能测试结果会受到设备性能、图片尺寸和 WebView 环境影响，因此测试报告需记录测试环境。

## 16. 风险分析

| 编号 | 风险 | 影响 | 应对措施 |
| --- | --- | --- | --- |
| RISK-01 | 大尺寸图片导致处理耗时超过 `300ms` | 影响性能评分 | 记录不同尺寸图片耗时，加入优化实现和测试说明 |
| RISK-02 | WebView 图片选择回调处理不完整 | 影响核心功能演示 | 优先实现并测试本地图片选择流程 |
| RISK-03 | 灰度公式或归一化逻辑偏离任务书 | 影响结果准确性 | 在测试用例中加入公式和边界检查 |
| RISK-04 | 文档声明超过实际实现 | 影响答辩可信度 | 文档中区分已实现、计划实现和可选扩展 |
| RISK-05 | 引入后端或网络依赖导致范围扩大 | 增加开发和验收风险 | 保持离线 APK 核心路径，不加入后端能力 |

## 18. 结论

本项目需求明确、范围可控，核心目标集中在移动端本地图像直方图生成与性能展示。系统应以 Android APK 形式交付，通过 Android WebView 加载 H5 Canvas 页面，在无后端、无数据库、无网络依赖的条件下完成本地图片选择、灰度化、统计、归一化、`256x100` 黑白直方图绘制和耗时展示。

后续实现与测试应优先保障核心验收路径稳定可靠，再考虑性能对比、测试数据整理和答辩展示增强。所有文档和演示材料应与实际源码和测试证据保持一致。
