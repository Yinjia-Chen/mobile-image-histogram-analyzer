## 1. 文档说明

### 1.1 文档目的

本文档用于说明“移动端图像直方图分析系统”的总体设计方案，包括系统架构、模块划分、主流程设计、数据结构设计、接口边界、界面设计、异常处理和部署方式。本文档承接《产物-需求分析报告.md》，为后续编码实现、测试计划、测试用例和使用说明提供设计依据。

| 原则 | 说明 |
| --- | --- |

## 2. 总体设计

### 2.1 系统定位

本系统是一个 Android WebView 壳 App。用户安装 APK 后，可以在手机本地选择图片，应用通过内置 H5 页面完成图片预览、灰度化、直方图统计、归一化、直方图绘制和耗时显示。

系统第一阶段重点实现主流程功能，不引入后端、登录、数据库、云存储或远程部署。性能优化相关内容仅预留扩展点，后续单独通过性能优化文档、测试记录和答辩材料进行说明。

### 2.2 总体架构

```text
Android APK
├── Android 原生壳层
│   ├── MainActivity
│   ├── WebView 初始化
│   ├── 本地 H5 资源加载
│   └── 本地图片选择支持
│
└── H5 Canvas 功能层
    ├── 页面交互模块
    ├── 图片预览模块
    ├── 像素读取模块
    ├── 灰度化与直方图统计模块
    ├── 归一化模块
    ├── 直方图绘制模块
    └── 状态与耗时展示模块
```

### 2.3 技术选型

| 层级 | 技术 | 用途 |
| --- | --- | --- |
| Android 壳层 | Android Studio、Java 或 Kotlin | APK 封装、WebView 容器、本地图片选择 |
| 页面层 | HTML、CSS | 移动端页面结构与样式 |
| 逻辑层 | JavaScript | 图片处理流程控制、灰度化、统计和绘制 |
| 图像处理 | H5 Canvas | 图片绘制、像素读取、直方图渲染 |
| 文档与协作 | Markdown、GitHub | 项目文档、版本管理和课程交付 |

## 4. 主流程设计

### 4.1 主流程说明

系统主流程如下：

```text
启动 APK
    ↓
WebView 加载本地 H5 页面
    ↓
用户选择本地图片
    ↓
页面展示原图预览
    ↓
Canvas 绘制图片
    ↓
读取 RGBA 像素数据
    ↓
按公式计算灰度值
    ↓
统计 256 个灰度 bin
    ↓
按最大值归一化到 0-100
    ↓
绘制 256x100 黑白直方图
    ↓
显示直方图和生成耗时
```

### 4.2 主流程步骤

| 步骤 | 输入 | 处理 | 输出 |
| --- | --- | --- | --- |
| S-01 启动应用 | 用户点击 App 图标 | Android 启动 MainActivity | 进入 WebView 容器 |
| S-02 加载页面 | 本地 H5 资源 | WebView 加载 `assets` 中的页面 | 显示主界面 |
| S-03 选择图片 | 用户操作 | 调用本地图片选择能力 | 图片文件或图片 URI |
| S-04 图片预览 | 图片数据 | 页面显示原图预览 | 可见图片预览 |
| S-05 读取像素 | 图片数据 | Canvas 绘制图片并读取 RGBA 数组 | 像素数组 |
| S-06 灰度计算 | RGBA 数组 | 按指定公式计算灰度值 | 灰度值序列 |
| S-07 直方图统计 | 灰度值序列 | 统计 256 个 bin | 直方图计数数组 |
| S-08 归一化 | 计数数组 | 按最大计数归一化到 `0-100` | 归一化数组 |
| S-09 绘制结果 | 归一化数组 | Canvas 绘制 `256x100` 黑白直方图 | 直方图图像 |
| S-10 展示耗时 | 处理过程计时 | 计算并显示耗时 | 耗时文本 |

## 5. 数据设计

### 5.1 输入数据

| 数据 | 来源 | 说明 |
| --- | --- | --- |
| 本地图片 | 手机相册或本地文件 | 用户选择的待处理图片 |
| RGBA 像素数组 | Canvas `getImageData` | 每个像素包含 R、G、B、A 四个通道 |

### 5.2 中间数据

| 数据结构 | 类型建议 | 说明 |
| --- | --- | --- |
| `gray` | Number | 单个像素转换后的灰度值，范围 `0-255` |
| `histogram[256]` | Array 或 TypedArray | 记录 256 个灰度值出现次数 |
| `maxCount` | Number | 当前图片中出现次数最多的灰度值计数 |
| `normalized[256]` | Array 或 TypedArray | 归一化到 `0-100` 的直方图高度数组 |
| `elapsedMs` | Number | 本次直方图生成耗时，单位 ms |

### 5.3 输出数据

| 输出 | 说明 |
| --- | --- |
| 原图预览 | 用户选择图片的页面预览 |
| 黑白直方图 | `256x100` 的黑白直方图 |
| 生成耗时 | 本次主流程处理耗时 |
| 状态提示 | 图片选择失败、读取失败等提示信息 |

## 6. 算法设计

### 6.1 灰度化公式

系统必须使用任务书指定公式：

```text
gray = red * 0.299 + green * 0.587 + blue * 0.114
```

实现时需要将计算结果映射到 `0-255` 的整数 bin。具体取整方式在编码阶段统一确定，并在测试用例中保持一致。

### 6.2 直方图统计

核心统计过程：

```text
初始化 histogram[256] 为 0
遍历图片中每个像素
    读取 red、green、blue
    按公式计算 gray
    将 gray 映射到 0-255 的整数 bin
    histogram[gray] += 1
```

设计约束：

- 必须保留 256 个灰度 bin。
- 每个有效像素都应参与统计。
- 不使用 CSS 伪造柱状图作为统计结果。
- 不改变灰度化公式。

### 6.3 归一化设计

归一化目标是将不同图片的像素数量分布映射到 `0-100` 的直方图高度范围。

```text
maxCount = max(histogram)
normalized[i] = histogram[i] / maxCount * 100
```

当图片读取失败或 `maxCount` 为 0 时，应停止绘制并显示异常提示。

### 6.4 直方图绘制

直方图输出尺寸为 `256x100`。设计方式如下：

- 横向每个像素列对应一个灰度值，范围 `0-255`。
- 纵向高度来自归一化结果，最大值为 100。
- 绘制颜色采用黑白方案。
- 页面展示时可以通过 CSS 放大显示，但源直方图数据仍应保持 `256x100` 语义。

## 8. Android 壳设计

### 8.1 Android 层职责

Android 层主要负责：

- 提供可安装 APK。
- 启动 MainActivity。
- 初始化 WebView。
- 加载本地 H5 资源。
- 支持本地图片选择。
- 保证无网络状态下主界面可加载。

### 8.2 H5 资源加载

第一阶段建议将 H5 文件放入 Android 项目的本地资源目录，例如：

```text
app/src/main/assets/
├── index.html
├── style.css
└── app.js
```

WebView 加载本地入口：

```text
file:///android_asset/index.html
```

### 8.3 图片选择交互

图片选择可以由 H5 的文件选择入口触发，并由 Android WebView 文件选择回调提供支持。第一阶段只要求完成本地图片选择和传入 H5 处理链路，不扩展云端选图或远程图片地址。

## 9. 异常处理设计

| 异常场景 | 处理方式 |
| --- | --- |
| 用户取消选图 | 不执行处理，页面提示“未选择图片”或保持初始状态 |
| 图片读取失败 | 停止处理并提示用户重新选择图片 |
| Canvas 获取像素失败 | 显示错误提示，不绘制错误结果 |
| 图片尺寸过大导致处理较慢 | 正常展示结果和耗时，后续性能文档中记录优化方向 |
| WebView 页面加载失败 | Android 层保留基本错误提示，便于定位资源路径问题 |

## 14. 小结

本概要设计围绕“离线 Android WebView APK + H5 Canvas 主流程”展开，将系统划分为 Android 壳层和 H5 功能层。第一阶段重点完成本地图片选择、图片预览、像素读取、灰度化、直方图统计、归一化、`256x100` 黑白直方图绘制和耗时展示，确保课程验收主路径清晰、稳定、可解释。

性能优化作为后续独立工作推进，不在本阶段扩大系统范围。该设计能够支撑后续编码实现、测试计划和答辩材料编写。
