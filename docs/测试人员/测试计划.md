# 移动端图像直方图分析系统测试计划

## 1. 文档说明

本文档从测试人员视角说明“移动端图像直方图分析系统”的测试范围、测试类型、测试环境、测试方法和退出标准。

本项目测试分为两类：

- 研发自测：以脚本、构建、资源扫描和确定性 fixture 为主，验证算法、离线边界和代码质量。
- 验收测试：以 Android 手机真实操作为主，验证 APK 安装、断网启动、本地选图、直方图生成、耗时展示和现场演示稳定性。

两类测试的口径不同。研发自测已经有较完整的仓库证据；验收测试仍需要补充手机型号、Android 版本、截图和实测耗时。

## 4. 测试类型

### 4.1 研发自测

| 类型 | 说明 | 主要命令或证据 |
| --- | --- | --- |
| 算法单元测试 | 验证公式、bin、归一化和绘制规则 | `npm run test:histogram` |
| 主流程 fixture | 使用 fake Canvas 跑完整生成流程 | `scripts/test-histogram-main-flow.cjs` |
| 离线资源检查 | 检查 assets、本地入口、无远程依赖、Manifest 权限 | `npm run test:offline` |
| 对照一致性 | 验证低效对照版本与当前优化版本输出一致 | `npm run test:baseline` |
| 性能 benchmark | 生成低效对照版本与当前优化版本同边界耗时对照 | `npm run benchmark:histogram` |
| 注释比例检查 | 验证核心源码注释覆盖要求 | `npm run check:source-comments` |
| harness 校验 | 汇总文档、算法、离线和源码注释检查 | `npm run harness:verify` |

### 4.2 验收测试

| 类型 | 说明 | 证据要求 |
| --- | --- | --- |
| APK 安装测试 | 在 Android 手机安装正式演示版本 APK | 手机型号、Android 版本、安装结果截图 |
| 断网启动测试 | 关闭网络后打开应用 | 断网状态说明、启动截图 |
| 图片选择测试 | 从本地相册或文件选择图片 | 测试图片格式、尺寸、操作结果 |
| 结果展示测试 | 查看原图预览和 `256x100` 直方图 | 结果截图 |
| 耗时测试 | 记录本次处理耗时 | 图片尺寸、耗时、是否低于 300ms |
| 性能对照测试 | 安装低效压力对照版本与正式演示版本，用同一图片对比 | 两个 APK 的耗时和结果说明 |
| 演示稳定性测试 | 按答辩路径完整走一遍 | 演示是否顺畅、是否需要备用方案 |

## 7. 测试策略

### 7.1 研发自测策略

研发自测优先保证规则正确：

1. 先用 fixture 验证灰度公式和 bin 统计。
2. 再验证主流程能从像素读取走到绘制。
3. 再检查 H5 assets 是否离线可用。
4. 最后用低效对照版本与当前优化版本对照说明性能策略。

这种测试更适合发现算法和代码层面的错误，不替代手机验收。

### 7.2 验收测试策略

验收测试优先保证现场可演示：

1. 使用正式演示版本 APK 作为主演示版本。
2. 先断网打开，证明离线能力。
3. 选择提前准备好的本地图片。
4. 展示预览、直方图和耗时。
5. 如时间允许，再打开低效压力对照版本 APK 做对比。

不要在现场临时选择超大图片，也不要把保存图片等增强功能放在主演示路径里。

## 9. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 真机信息缺失 | 测试报告不完整 | 补充手机型号、Android 版本、截图和耗时 |
| 图片过大导致耗时波动 | 影响 300ms 说明 | 准备中等尺寸演示图，大图只作补充观察 |
| 低效对照版本与当前优化版本口径不同 | 性能对照不可信 | 使用同边界 benchmark 和一致性测试 |
| 现场网络图片不可用 | 演示失败 | 只使用本地图片，不依赖网络 |
| 保存图片功能受系统限制 | 辅助功能失败 | 不把保存功能放入核心验收路径 |

## 10. 计划结论

本项目测试计划以“研发自测证明算法正确，验收测试证明 APK 可演示”为主线。当前研发自测证据充分，验收测试已经具备执行条件，但最终报告仍需要补充真机环境、截图和耗时数据。
