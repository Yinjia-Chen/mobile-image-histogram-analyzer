## 5. 算法测试结论

| 规则 | 测试结论 |
| --- | --- |
| 灰度公式 | 使用 `gray = red * 0.299 + green * 0.587 + blue * 0.114`，fixture 通过 |
| 取整规则 | 使用 `Math.round`，测试口径一致 |
| 256 bins | 统计数组长度为 256 |
| 像素总数 | bins 求和等于输入像素数 |
| 归一化 | 高度范围为 `0-100` |
| 绘制尺寸 | 输出语义为 `256x100` |

算法测试结论：通过。

## 6. 离线测试结论

离线检查覆盖以下内容：

- `index.html` 引用本地 CSS 和 JS；
- H5 页面包含 `type="file"` 与 `accept="image/*"`；
- 直方图 Canvas 源尺寸为 `256x100`；
- `MainActivity` 加载 `file:///android_asset/index.html`；
- Android 文件选择回调存在；
- Manifest 未申请网络权限；
- APK 内包含本地 assets。

离线测试结论：研发自测通过。真机断网测试已有用户反馈通过，但完整设备和截图记录仍需回填。

## 7. 性能测试结论

当前已有开发机 benchmark：

| Case | Image size | Pixels | Comparison baseline | Current optimized | Speedup | 当前实现是否低于 300ms |
| --- | --- | ---: | ---: | ---: | ---: | --- |
| `small-demo` | 320x180 | 57,600 | 46.22 ms | 1.42 ms | 32.5x | 是 |
| `phone-mid` | 800x600 | 480,000 | 407.1 ms | 2.30 ms | 176.9x | 是 |
| `phone-large` | 1280x720 | 921,600 | 700.4 ms | 1.99 ms | 351.6x | 是 |

说明：

- 该表来自开发机 Node.js 合成 RGBA 数据；
- 它证明实现策略差异，不替代 Android 手机实测；
- baseline 与 optimized 的计时边界保持一致；
- 两种模式必须输出一致结果，否则测试失败。

性能测试结论：研发 benchmark 通过，真机耗时表待补。
