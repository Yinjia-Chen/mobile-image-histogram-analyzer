# 移动端图像直方图分析系统测试用例

## 1. 文档说明

本文档列出移动端图像直方图分析系统的测试用例。用例分为两组：

- `ST`：研发自测用例，主要通过脚本和本地构建验证。
- `AT`：验收测试用例，主要通过 Android 手机真实操作验证。

## 2. 研发自测用例

| 用例编号 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 | 证据 |
| --- | --- | --- | --- | --- | --- |
| ST-01 | 灰度公式测试 | 已安装 Node.js | 运行 `npm run test:histogram` | 指定 RGB 输入按 `gray = red * 0.299 + green * 0.587 + blue * 0.114` 得到预期 bin | `scripts/test-histogram-algorithm.cjs` |
| ST-02 | 256 bin 统计测试 | 同 ST-01 | 运行 `npm run test:histogram` | bins 长度为 256，覆盖 `0-255` | `scripts/test-histogram-algorithm.cjs` |
| ST-03 | 像素总数一致性测试 | 同 ST-01 | 运行 `npm run test:histogram` | bins 求和等于输入像素数 | `scripts/test-histogram-main-flow.cjs` |
| ST-04 | 归一化测试 | 同 ST-01 | 运行 `npm run test:histogram` | 最大 bin 高度为 100，其余高度在 `0-100` | `scripts/test-histogram-algorithm.cjs` |
| ST-05 | `256x100` 绘制测试 | 同 ST-01 | 运行 `npm run test:histogram` | 输出 Canvas 语义为 `256x100`，黑白柱按底部向上绘制 | `scripts/test-histogram-main-flow.cjs` |
| ST-06 | 主流程 fake Canvas 测试 | 同 ST-01 | 运行 `npm run test:histogram` | 生成结果包含 bins、normalized、elapsedMs 和绘制结果 | `scripts/test-histogram-main-flow.cjs` |
| ST-07 | 离线 assets 检查 | H5 assets 已存在 | 运行 `npm run test:offline` | 页面引用本地 CSS/JS，无远程资源、无网络权限依赖 | `scripts/test-offline-assets.cjs` |
| ST-08 | baseline 一致性测试 | baseline 脚本已存在 | 运行 `npm run test:baseline` | baseline 与 optimized 的 bins、归一化和渲染缓冲区一致 | `scripts/test-histogram-baseline.cjs` |
| ST-09 | 性能 benchmark | benchmark 脚本已存在 | 运行 `npm run benchmark:histogram` | 输出 small、mid、large 三组同边界耗时对照 | `scripts/benchmark-histogram-performance.cjs` |
| ST-10 | 源码注释检查 | app/src 源码存在 | 运行 `npm run check:source-comments` | 核心源码注释比例满足项目要求，覆盖文件无缺失 | `scripts/check-source-comments.cjs` |
| ST-11 | harness 综合校验 | 项目脚本齐全 | 运行 `npm run harness:verify` | 文档、算法、离线、注释检查均通过 | `scripts/harness-verify.sh` |

## 3. 验收测试用例

| 用例编号 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 | 证据要求 |
| --- | --- | --- | --- | --- | --- |
| AT-01 | 优化版 APK 安装 | Android 手机可用 | 安装 `dist/mobile-histogram-optimized-coinstall-debug.apk` | 安装成功，桌面出现应用图标 | 手机型号、Android 版本、安装截图 |
| AT-02 | 断网启动 | 已安装优化版 APK | 关闭 Wi-Fi 和移动数据，打开应用 | 应用进入主界面，无网络也能加载页面 | 断网状态截图、启动截图 |
| AT-03 | 本地图片选择 | 应用已启动，手机有图片 | 点击“接入图像”，选择本地图片 | 系统返回应用，开始处理图片 | 操作截图或录屏 |
| AT-04 | 原图预览 | 已选择图片 | 查看接入页预览区域 | 显示所选原图 | 预览截图 |
| AT-05 | 直方图生成 | 已选择图片 | 切换到输出页 | 显示 `256x100` 黑白直方图 | 输出页截图 |
| AT-06 | 耗时展示 | 已生成直方图 | 查看指标或输出区域耗时 | 显示本次生成耗时，单位为 ms | 耗时截图、图片尺寸 |
| AT-07 | 指标说明 | 已生成直方图 | 切换到指标页 | 显示 bins、像素总数、最大计数、性能余量等信息 | 指标页截图 |
| AT-08 | 协议说明 | 应用可用 | 切换到协议页 | 可看到灰度公式、256 bins、归一化和 `256x100` 规则说明 | 协议页截图 |
| AT-09 | 重新选择图片 | 已处理过一张图 | 再次选择另一张图片 | 页面刷新为新图片和新直方图 | 前后对比截图 |
| AT-10 | 异常取消选择 | 应用可用 | 打开选择器后取消 | 应用不崩溃，保持可操作 | 操作记录 |
| AT-11 | 优化前 APK 安装 | 手机可安装第二个 APK | 安装 `dist/mobile-histogram-baseline-coinstall-debug.apk` | 与优化版可并装 | 桌面两个图标截图 |
| AT-12 | 优化前后性能对比 | 两个 APK 均安装 | 使用同一图片分别生成直方图 | 结果一致，耗时可对比 | 两个版本耗时截图 |
| AT-13 | 直方图放大 | 已生成直方图 | 点击直方图 | 横向放大显示，不改变结果数据 | 放大截图 |
| AT-14 | 长按保存拼接图 | 已生成直方图 | 长按直方图并确认保存 | 系统尝试保存拼接 PNG，失败时有提示 | 保存结果截图，可选 |
| AT-15 | 演示路径完整性 | 准备好测试图片 | 按答辩路径完整操作一次 | 3 至 5 分钟内可完整展示核心功能 | 演示记录 |

## 4. 验收图片建议

| 图片编号 | 类型 | 建议尺寸 | 用途 |
| --- | --- | --- | --- |
| IMG-01 | 普通照片 | 约 800x600 | 主演示图片 |
| IMG-02 | 偏亮或偏暗图片 | 任意常见尺寸 | 解释直方图分布 |
| IMG-03 | 较大图片 | 约 1280x720 或更高 | 性能观察，不建议作为首个演示图 |

## 5. 通过标准

### 5.1 研发自测通过标准

- `npm run test:histogram` 通过；
- `npm run test:offline` 通过；
- `npm run test:baseline` 通过；
- `npm run harness:verify` 通过；
- benchmark 能输出同边界对照表。

### 5.2 验收测试通过标准

- 优化版 APK 可安装和启动；
- 断网后仍可打开本地页面；
- 可选择本地图片并显示预览；
- 可生成 `256x100` 黑白直方图；
- 可显示生成耗时；
- 指标页能支撑公式、bins、归一化和性能说明；
- 至少保留一组真机截图和耗时记录。
