## 2. 项目背景

图像直方图是观察图像亮度分布的一种基础方法。对一张彩色图片而言，先把每个像素转换为灰度值，再统计 `0-255` 各灰度值出现的次数，就能得到图片整体偏亮、偏暗、对比度是否集中等信息。

课程题目要求开发一个移动端程序，能够选择或上传图像，统计像素并生成 `256x100` 黑白直方图，同时显示直方图生成速度。项目最终选择 Android WebView 壳 App + H5 Canvas 的实现路线：Android 层负责 APK 和手机本地能力，H5 层负责页面、Canvas 图像处理和结果展示。

这个选型的好处是边界清楚、离线可运行，也便于在短周期课程项目中完成“能安装、能演示、能解释、能测试”的交付闭环。

## 5. 业务范围

### 5.1 范围内

| 类别 | 需求内容 |
| --- | --- |
| 移动端交付 | 以 Android APK 作为最终交付形态 |
| 本地运行 | WebView 加载 APK 内置 H5 assets，不依赖远程页面 |
| 图片输入 | 用户从手机本地选择一张图片 |
| 图片预览 | 页面展示已选择图片，方便确认输入 |
| 像素读取 | 使用 Canvas 获取 RGBA 像素数据 |
| 灰度计算 | 使用课程指定公式计算灰度值 |
| 直方图统计 | 统计 256 个灰度值出现次数 |
| 归一化 | 将计数按最大值缩放到 `0-100` |
| 结果绘制 | 绘制 `256x100` 黑白直方图 |
| 耗时展示 | 显示本次直方图生成耗时 |
| 性能说明 | 提供低效对照版本与当前优化版本的同边界对照 |

### 5.2 范围外

| 范围外内容 | 不纳入原因 |
| --- | --- |

| 云端图片上传和存储 | 项目强调本地离线处理 |

| 多图批量处理 | 当前验收主路径是一张图片生成一个直方图 |

### 6.5 FR-05 灰度直方图生成

| 项目 | 内容 |
| --- | --- |
| 描述 | 系统读取图片像素，计算灰度值，统计 256 个 bin |
| 输入 | 图片 RGBA 像素数据 |
| 输出 | 长度为 256 的灰度统计数组 |
| 优先级 | 高 |
| 验收标准 | bin 总数为 256，bin 求和等于像素总数 |
| 证据 | `scripts/test-histogram-algorithm.cjs`、`scripts/test-histogram-main-flow.cjs` |

核心公式：

```text
gray = red * 0.299 + green * 0.587 + blue * 0.114
```

## 8. 数据需求

### 8.1 输入数据

| 数据 | 来源 | 说明 |
| --- | --- | --- |
| 本地图片文件 | 手机相册或文件系统 | 用户选择的 JPEG、PNG 等常见图片 |
| RGBA 像素数据 | Canvas `getImageData` | 每个像素包含 R、G、B、A 四个通道 |

### 8.2 中间数据

| 数据 | 类型 | 说明 |
| --- | --- | --- |
| `gray` | Number | 单个像素的灰度值 |
| `bins` | `Uint32Array(256)` | 256 个灰度值的计数 |
| `maxCount` | Number | 当前图片中最大灰度计数 |
| `normalized` | Array 或 TypedArray | 归一化到 `0-100` 的高度 |
| `elapsedMs` | Number | 本次处理耗时 |

### 8.3 输出数据

| 数据 | 说明 |
| --- | --- |
| 原图预览 | 用户选择图片的页面展示 |
| 黑白直方图 | `256x100` Canvas 输出 |
| 生成耗时 | 本次处理耗时，单位 ms |
| 指标说明 | bins 总数、像素总数、最大计数、性能余量等 |
| 可选拼接图 | 用户长按确认后保存的“原图 + 说明 + 直方图”图片 |
