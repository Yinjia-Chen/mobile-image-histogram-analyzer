# 移动端图像直方图分析系统使用说明

## 1. 文档说明

本文档面向实际使用和课程验收演示，说明 APK 安装、启动、选图、查看直方图、查看指标和优化前后对比的方法。

当前仓库已输出优化版和优化前对照版 APK。最终提交前，建议补充一组真机截图和耗时数据，用于测试报告和答辩 PPT。

## 2. 使用前准备

### 2.1 设备准备

| 项目 | 要求 |
| --- | --- |
| 设备 | Android 手机或 Android 模拟器 |
| 系统 | 建议 Android 6.0 及以上 |
| 网络 | 核心功能不需要网络，建议验收时断网演示 |
| 图片 | 手机本地准备 1 至 2 张测试图片，建议包含一张中等尺寸图片 |

### 2.2 APK 文件

| APK | 路径 | 用途 |
| --- | --- | --- |
| 优化后版本 | `dist/mobile-histogram-optimized-coinstall-debug.apk` | 正式演示版本 |
| 优化前版本 | `dist/mobile-histogram-baseline-coinstall-debug.apk` | 性能对照版本 |
| 阶段 4 基础版本 | `dist/mobile-histogram-stage4-debug.apk` | 基础主流程交付记录 |

一般演示优先安装“优化后版本”。只有需要讲性能优化时，再安装“优化前版本”做对比。

## 3. 安装说明

1. 将 APK 文件复制到 Android 手机。
2. 在手机文件管理器中点击 APK。
3. 如果系统提示“允许安装未知来源应用”，按系统提示授权当前文件管理器或浏览器。
4. 完成安装后，在桌面找到应用图标。
5. 点击图标打开应用。

如果安装失败，先检查：

- APK 文件是否完整；
- 手机是否允许安装未知来源应用；
- Android 版本是否过低；
- 手机剩余空间是否足够；
- 是否安装了同包名旧版本，需要先卸载旧版本。

## 4. 快速演示流程

推荐答辩或验收时使用下面这条路径：

```text
打开优化版 APK
-> 关闭网络或说明当前为离线运行
-> 点击“接入图像”
-> 选择本地测试图片
-> 查看原图预览
-> 切换到“输出”查看 256x100 直方图
-> 切换到“指标”说明耗时、像素总数、bins 和性能余量
```

这条流程覆盖课程要求的核心点：移动端、选图、灰度直方图、`256x100` 输出、耗时展示、离线运行。

## 5. 功能使用说明

### 5.1 打开应用

安装完成后点击桌面图标，应用会进入本地 H5 页面。页面由 Android WebView 加载 APK 内置资源，不需要访问服务器。

如果启动时看到短暂开屏动画，这是正常的界面反馈，不代表已经生成直方图。真实直方图只有在用户选择图片后才会生成。

### 5.2 选择图片

1. 进入“接入”页面。
2. 点击图片选择按钮。
3. 从相册或文件管理器中选择一张图片。
4. 等待页面显示原图预览和处理状态。

建议演示时提前准备图片，避免现场临时寻找文件影响节奏。

### 5.3 查看原图预览

选图后，页面会显示当前图片预览。预览的作用是确认输入图片，不参与最终直方图尺寸判断。

直方图尺寸始终以输出 Canvas 的 `256x100` 为准，不以预览图片显示大小为准。

### 5.4 查看直方图

切换到“输出”页面，可以看到生成的黑白直方图。

读图方式：

| 方向 | 含义 |
| --- | --- |
| 横向 | 从左到右对应灰度值 `0-255` |
| 纵向 | 当前灰度值像素数量归一化后的高度 |
| 黑色柱 | 对应灰度值的相对数量 |
| 白色背景 | 没有被柱子覆盖的区域 |

直方图源尺寸为 `256x100`，页面展示时可能为了手机屏幕适配做视觉放大，但统计语义不变。

### 5.5 查看指标

切换到“指标”页面，可以查看：

- 生成耗时；
- 256 个 bins；
- 图片像素总数；
- 最大灰度计数；
- 归一化结果；
- `300ms` 性能目标相关说明；
- baseline 与 optimized 的一致性说明。

这些指标用于答辩时解释“结果为什么是对的”和“性能为什么可说明”。

### 5.6 查看协议说明

“协议”页面用于说明固定规则：

```text
gray = red * 0.299 + green * 0.587 + blue * 0.114
```

同时说明 256 bins、`0-100` 归一化和 `256x100` 输出规则。验收时如果老师追问算法口径，可以直接用这一页辅助说明。

### 5.7 查看日志

“日志”页面显示处理过程中的关键步骤，例如选图、解码、Canvas 读取、灰度统计、归一化、渲染等。日志主要用于解释流程，不需要用户手动修改。

### 5.8 放大直方图

在结果页点击直方图，可以横向放大查看。该功能用于现场观察结果，不改变直方图源数据。

### 5.9 保存结果图

长按直方图后，系统会弹出保存确认。点击确认后，应用会尝试保存“原图 + 分割说明 + 直方图”的拼接 PNG 到系统图片库。

注意：

- 保存功能是辅助功能，不是课程核心验收路径；
- 部分 Android 系统可能受媒体库或权限策略影响；
- 如果保存失败，不影响选图、计算和直方图展示。

## 6. 优化前后对比说明

如果需要演示性能优化：

1. 安装优化前 APK：`mobile-histogram-baseline-coinstall-debug.apk`。
2. 安装优化后 APK：`mobile-histogram-optimized-coinstall-debug.apk`。
3. 使用同一张图片分别生成直方图。
4. 比较两边显示的耗时和结果一致性。

说明口径：

- 优化前版本是低效压力对照，用于说明对象分配、重复计算和重复扫描的性能影响；
- 优化后版本是正式演示版本；
- 两个版本应保持灰度公式、256 bins、归一化和 `256x100` 输出一致；
- 开发机 benchmark 可说明策略差异，正式报告仍建议补真机耗时。

## 7. 常见问题

| 问题 | 可能原因 | 处理方式 |
| --- | --- | --- |
| APK 无法安装 | 未允许未知来源、文件损坏、系统版本过低 | 重新复制 APK，打开安装授权，或换测试设备 |
| 打开后没有图片 | 还未选择图片 | 点击接入图像选择本地图片 |
| 选择图片后无结果 | 图片读取失败或格式异常 | 换一张 JPEG/PNG 图片重试 |
| 耗时波动 | 手机负载、图片尺寸、系统调度不同 | 使用同一图片多测几次，记录平均或典型值 |
| 直方图看起来很扁 | 图片灰度分布集中或归一化后差异小 | 查看指标页的最大计数和分布说明 |
| 保存图片失败 | 系统权限或媒体库策略限制 | 使用截图替代，核心功能不受影响 |
| 断网还能用吗 | 可以 | 核心页面和脚本已打包在 APK 内 |

## 8. 演示建议

正式答辩建议准备：

- 一台已安装优化版 APK 的 Android 手机；
- 一张中等尺寸测试图片；
- 一张偏亮或偏暗的图片，用于说明直方图分布；
- 可选安装优化前 APK；
- 一张结果页或指标页截图；
- 备用 APK 文件。

现场演示不要临时选择超大原图，避免图片太大导致耗时波动，也不要把保存图片功能放在主演示路径里。

## 9. 验收检查清单

| 检查项 | 是否必须 | 说明 |
| --- | --- | --- |
| APK 可启动 | 是 | 移动端交付基础 |
| 断网可打开页面 | 是 | 证明离线运行 |
| 可选择本地图片 | 是 | 输入要求 |
| 可展示原图预览 | 是 | 便于确认输入 |
| 可生成直方图 | 是 | 核心输出 |
| 直方图为 `256x100` | 是 | 课程明确要求 |
| 显示耗时 | 是 | 性能要求 |
| 指标页可解释结果 | 建议 | 答辩加分 |
| 优化前后对比 | 建议 | 说明性能优化 |
| 保存拼接图 | 可选 | 辅助展示 |

## 10. 使用说明结论

本系统的使用路径较短，适合课程现场演示。普通用户只需要完成“打开应用、选择图片、查看直方图”三步；答辩时再补充指标页、协议页和优化前后对比即可。

最终提交前建议把实际手机型号、Android 版本、测试图片尺寸、一次典型耗时和截图补入测试报告。
