# 移动端图像直方图分析系统

本项目是软件开发实践课程选题 **“图像直方图计算及性能优化”** 的实现仓库。系统采用 **Android WebView 壳 App + H5 Canvas** 的混合开发方式，最终交付 Android APK，在手机本地完成图片灰度化、直方图统计、直方图绘制和耗时展示。

项目核心目标是：在移动端离线环境下，对本地图片生成 `256x100` 黑白直方图，并显示本次生成耗时。`300ms` 作为后续性能优化阶段的参考目标，性能优化将单独记录和推进。

## 项目背景

图像直方图是描述图像亮度分布的重要工具，可以反映一张图片中不同灰度值的像素数量分布情况。通过直方图，用户可以快速判断图片整体偏亮、偏暗、对比度是否集中等基本特征。

课程任务书要求开发一个移动端程序，上传或选择图像后统计彩色图像中的像素，并生成 `256x100` 的黑白直方图，同时显示直方图生成速度。该任务既关注结果准确性，也关注移动端环境下的计算性能，因此适合作为图像处理、移动端开发和性能优化的综合实践项目。

结合团队对开发难度、验收稳定性和答辩展示效果的判断，本项目选择官方课题 1，并采用 Android WebView + H5 Canvas 的技术路线。该方案既可以交付真实 APK，又能将核心算法集中在前端 Canvas 和 JavaScript 中，便于快速实现、测试和优化。

## 项目目标

- 完成任务书要求的 `256x100` 黑白直方图生成。
- 使用指定灰度化公式，保证统计结果具有一致性。
- 在手机本地完成图像处理，减少网络依赖并保护用户图片隐私。
- 展示直方图生成耗时，支撑基础测试和答辩说明。
- 先保证主流程功能稳定，性能优化后续单独记录和推进。
- 形成可安装、可演示、可测试、可答辩的课程实践作品。

## 核心功能

| 序号 | 功能模块 | 功能说明 |
| --- | --- | --- |
| 1 | 图片选择 | 从手机本地相册或文件中选择待分析图片 |
| 2 | 原图预览 | 在应用界面中展示用户选择的图片 |
| 3 | 灰度化处理 | 按题目指定公式将彩色像素转换为灰度值 |
| 4 | 直方图统计 | 统计 `0-255` 共 256 个灰度值的像素出现次数 |
| 5 | 结果归一化 | 将统计结果按最大像素数量等比例归一化到 `0-100` |
| 6 | 直方图绘制 | 使用 Canvas 绘制 `256x100` 黑白直方图 |
| 7 | 耗时显示 | 展示直方图生成耗时，用于验证计算性能 |
| 8 | 后续优化预留 | 主流程稳定后再单独记录性能优化过程 |

## 灰度化公式

本项目使用任务书指定的灰度化公式：

```text
gray = red * 0.299 + green * 0.587 + blue * 0.114
```

其中 `red`、`green`、`blue` 分别表示像素的 RGB 通道值。计算出的灰度值会归入 `0-255` 的统计数组中，形成原始灰度直方图数据。

## 图像处理流程

```text
选择本地图片
    ↓
Canvas 绘制图片
    ↓
读取 RGBA 像素数据
    ↓
按指定公式计算灰度值
    ↓
统计 0-255 灰度值出现次数
    ↓
按最大计数归一化到 0-100
    ↓
绘制 256x100 黑白直方图
    ↓
展示耗时与结果
```

## 技术路线

```text
Android APK
└── Android WebView 容器
    └── H5 Canvas 页面
        ├── 图片选择与预览
        ├── 像素读取
        ├── 灰度化计算
        ├── 直方图统计
        ├── 直方图绘制
        └── 耗时展示
```

Android 原生层负责应用封装、WebView 初始化、本地页面加载和图片选择能力。H5 层负责页面展示、Canvas 图像处理和直方图算法实现。

## 离线验收说明

本项目不依赖后端服务、数据库或远程接口。APK 安装到 Android 手机后，只要手机本地准备好测试图片，即使没有网络，也可以完成以下验收流程：

1. 打开应用。
2. 选择手机本地图片。
3. 查看原图预览。
4. 生成 `256x100` 黑白直方图。
5. 查看直方图生成耗时。
